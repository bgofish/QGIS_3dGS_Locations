Gaussian Splat Viewer

A QGIS 3.x plugin for importing, georeferencing, and visualizing 3D Gaussian Splatting scenes on the QGIS map canvas.

Workflow

  1. Import — Load a .ply or .splat file.
  2. Georeference — Set the CRS, real-world origin, scale, and bearing.
  3. Visualize — Render the scene to a GeoTIFF raster layer.

Supported File Formats

ExtensionFormatNotes
.ply3DGS standard PLYProduced by gaussian-splatting, LichtFeld Studio, etc.
.splatBinary packedantimatter15 / SuperSplat viewer format (32 bytes/gaussian)

Georeferencing Parameters

ParameterDescription
CRSOutput coordinate reference system (e.g. EPSG:2193 NZTM, EPSG:4326 WGS84)
Origin X / YThe geographic coordinate that corresponds to the scene's local (0,0,0)
ScaleMetres per scene unit. If your scene was captured in metres, use 1.0
Bearing correctionClockwise degrees from north to the scene's +X axis
Z offsetVertical shift in metres applied after scaling

Render Settings

SettingDescription
ResolutionPixel size in CRS units. Smaller = sharper but slower.
Max gaussiansSubsample cap for performance. 200k is a good start.
View axisZ = top-down (default for maps). X/Y for elevation sections.
Performance tip: For scenes with >500k gaussians, increase the resolution value (coarser pixels) or reduce Max gaussians until you get a feel for the georeferencing, then do a final high-res render.

Known Limitations

  • Rendering is CPU-based. Very large scenes (>1M gaussians) may take several minutes.
  • The 2D canvas visualizer projects in orthographic top-down view. Perspective rendering requires the QGIS 3D Map View (future work).
  • Spherical harmonics above DC term are not used for canvas rendering (colour is DC-only).
